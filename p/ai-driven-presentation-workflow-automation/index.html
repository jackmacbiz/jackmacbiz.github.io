This guide provides a proprietary, multi-agent workflow designed to transform raw, unstructured data into high-fidelity, boardroom-ready presentations. The AI Presentation Stack: Essential Tools & Integrations

To achieve an 80% reduction in creation time, you cannot rely on a single LLM. You need a specialized stack that handles different cognitive loads.

1. **The Brain (Orchestrator):** **Claude 3.5 Sonnet.** While GPT-4o is excellent for general reasoning, Claude 3.5 Sonnet currently outperforms in large-context window management and following complex, multi-step JSON schemas without "breaking character."

2. **The Researcher (Data Ingestion):** **Perplexity AI (Pro Mode).** Use this to ingest live web data and academic citations. It provides the verifiable "source of truth" required for corporate decks.

3. **The Architect (Structure & Code):** **ChatGPT (GPT-4o) with Advanced Data Analysis enabled.** This is your engine for generating Python scripts to manipulate `.pptx` files directly, bypassing the limitations of VBA.

4. **The Visual Designer:** **Midjourney v6.1.** Essential for generating high-end, consistent photographic and iconographic assets.

5. **The Automation Bridge:** **Python (python-pptx library).** Do not rely on VBA for complex layouts. Python allows for programmatic control over text boxes, image placement, and shape scaling that VBA lacks.

Phase 1: Automated Research & Data Extraction Strategies

The biggest failure in AI presentations is "hallucinated insight." To prevent this, you must implement a **Two-Pass Extraction Strategy.**

Step 1: The Deep Scan (Perplexity)

Upload your raw PDFs, earnings reports, or meeting transcripts to Perplexity. Use this prompt to extract structured metrics:

> **Prompt:** "Act as a Senior Market Analyst. Analyze the attached [Document Name]. Extract all quantitative metrics, specifically focusing on Year-over-_Year (YoY) growth, market share percentages, and competitive threats. Format the output as a Markdown table. For every metric, include a direct quote from the text as a 'Source Verification' column."

Step 2: The Structured Synthesis (Claude 3.5 Sonnet)

Once you have the Markdown table, pass it to Claude to transform it into a "Slide Logic Map." This prevents the AI from losing the narrative thread.

> **Prompt:** "I am providing a Markdown table of market data. Your task is to convert this data into a 'Slide Narrative Map.' For each slide, define: 1. The Core Assertion (The 'So What?'), 2. The Supporting Data Point (from the table), and 3. The Visual Instruction (e.g., 'Bar chart showing X vs Y'). Do not invent data. If a metric is missing, mark it as [DATA GAP]."

Phase 2: Prompt Engineering for Narrative Storytelling & Scripting

A deck is not a list of facts; it is a persuasive argument. You must use the **SCQA Framework (Situation, Complication, Question, Answer)** to prompt your LLM.

The Master Scripting Template

Use this template to generate the "Speaker Notes" and "Slide Text" simultaneously. This ensures the visual text is punchy (minimalist) while the notes are deep (informative).

**The Prompt Schema:**

```markdown

Context: [Insert Project Overview]

Framework: SCQA (Situation, Complication, Question, Answer)

Tone: [e.g., Urgent, Authoritative, Visionary]

Task: Generate a 10-slide presentation outline.

For each slide, output in the following JSON format:

{

"slide_number": integer,

"slide_title": "string (max 7 words)",

"bullet_points": ["string", "string", "string"],

"speaker_notes": "string (detailed, conversational)",

"visual_layout": "string (e.g., split_screen, centered_icon, full_bleed_image)"

}

Constraints:

  • Bullet points must follow the 'One Idea Per Line' rule.
  • No more than 3 bullets per slide.
  • All data must be pulled from the provided context.

    • ```

    Phase 3: Automating Slide Layouts with Python Workflows

    VBA is fragile and prone to crashing when parsing JSON. Instead, use **Python** and the `python-pptx` library. This allows you to feed the JSON output from Phase 2 directly into a script that builds the `.pptx` file.

    The Bridge: From JSON to PPTX

    First, ensure your LLM output is strictly valid JSON. Then, run this Python script locally.

    **The Python Automation Script:**

    ```python

    import json

    from pptx import Presentation

    from pptx.util import Inches, Pt

    Load your JSON data generated by Claude/GPT

    json_data = '''

    [

    {

    "slide_number": 1,

    "slide_title": "Q3 Market Expansion Strategy",

    "bullet_points": ["Targeting EMEA Region", "Focus on SaaS Verticals", "Projected 15% Growth"],

    "visual_layout": "title_only"

    _},

    {

    "slide_number": 2,

    "slide_title": "Competitive Landscape",

    "bullet_points": ["Competitor A: 20% Share", "Competitor B: 15% Share", "Our Position: Emerging"],

    "visual_layout": "content"

    }

    ]

    '''

    slides_data = json.loads(json_data)

    prs = Presentation()

    for slide_info in slides_data:

    # Use 'Title and Content' layout (Layout index 1)

    slide_layout = prs.slide_layouts[1]

    slide = prs.slides.add_slide(slide_layout)

    # Set Title

    title = slide.shapes.title

    title.text = slide_info['slide_title']

    # Set Content

    body_shape = slide.placeholders[1]

    tf = body_shape.text_frame

    tf.text = slide_info['bullet_points'][0]

    for point in slide_info['bullet_points'][1:]:

    p = tf.add_paragraph()

    p.text = point

    p.level = 0

    prs.save('Automated_Presentation.pptx')

    print("Presentation generated successfully.")

    ```

    Phase 4: Generative AI for Consistent Visual Assets & Iconography

    To avoid the "AI-generated" look (which uses generic, glossy 3D characters), you must use **Style References (--sref)** in Midjourney. This ensures that a slide from page 1 looks like it belongs to page 10.

    Achieving Visual Cohesion

    1. **The Anchor Image:** Generate one high-quality "Hero" image that represents your brand's aesthetic.

    2. **The `--sref` Technique:** Copy the URL of your Hero Image.

    3. **The Prompt Construction:**

    `[Subject Description] --sref [URL of Hero Image] --sw 1000 --v 6.1`

    **Example Workflow for a Tech Startup:**

    * **Step 1 (The Hero):** `/imagine prompt: A minimalist, high-tech macro shot of blue circuit patterns, cinematic lighting, 8k --ar 16:9` (Copy this image URL).

    * **Step 2 (The Icon):** `/imagine prompt: A 3D isometric icon of a cloud server, frosted glass texture --sref [URL] --v 6.1`

    * **Step 3 (The Background):** `/imagine prompt: Abstract flowing silk texture, deep navy and silver --sref [URL] --v 6.1`

    **Parameter Guide:**

    * `--sw 1000`: Sets "Style Weight" to maximum. Use this to force the AI to adhere strictly to your anchor image.

    * `--cref [URL]`: Use "Character Reference" if you are using a consistent person/mascot across slides.

    Phase 5: The Human-in-the-Loop: Rapid Review Protocols

    Never present an unvetted AI deck. Use this **3-Point Audit**:

    1. **The Data Integrity Check:** Cross-reference the "Slide Logic Map" against the original source PDF. Focus specifically on decimal points and percentages.

    2. **The "Hallucination" Scan:** Search the Speaker Notes for "certainly," "it is widely believed," or "as of the latest trends." These are linguistic markers where LLMs often pivot from fact to speculation.

    3. **The Visual Friction Test:** Flip through the slides rapidly. If any slide feels "heavier" (too much text) or "emptier" than others, adjust the Python JSON structure and re-run.

    Troubleshooting: Common Pipeline Failures

    | Failure | Cause | Solution |

    | :--- | :--- | :--- |

    | **JSON Parsing Error** | LLM added conversational text (e.g., "Here is your JSON:") | Use "Output ONLY valid JSON" in your prompt. Use a Python `try-except` block to strip non-JSON characters. |

    | **Python `pptx` Error** | Missing keys in the JSON (e.g., `visual_layout` is missing) | Implement a **JSON Schema Validator** (using the `jsonschema` library) before running the PPTX script. |

    | **Visual Inconsistency** | Midjourney `--sref` weight is too low | Increase `--sw` to 1000. Ensure the reference image is high-contrast and color-dominant. |

    | **Broken Layouts** | Text overflow in Python-generated text boxes | In your Python script, add a character count check. If `len(text) > 200`, trigger a `font_size` reduction function. |
